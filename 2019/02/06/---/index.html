安装

基本介绍

es是一个基于Apache Lucene(TM)的开源搜索引擎。无论在开源还是专有领域，Lucene可以被认为是迄今为止最先进、性能最好的、功能最全的搜索引擎库，在企业中得到了广泛的应用。

  1. 分布式的实时文件存储，每个字段都被索引并可被搜索
  2. 分布式的实时分析搜索引擎
  3. 可以扩展到上百台服务器，处理PB级结构化或非结构化数据

说明：

1. curl是命令行 http请求的方式，es支持http请求和api接口方式请求
2. localhost:9200是本地的es默认的安装地址与端口
3. bank是索引_index的名称，account是类型_type的名称，1是文档的_id，
4. pretty是将返回的信息以可读的JSON形式返回。
5. 文档id可以不指定，系统自动生成唯一的id，推介系统后者，特别在大批量导入的时候，因为指定id每次都会校验id是否已经存在。
6. POST命令插入文档时，如果ID已经存在的话，会执行update更新文档。返回结果中_version字段表明当前文档的版本。
7. 可以使用PUT命令来校验id是否存在 curl -XPUT 'localhost:9200/bank/account/?op_type=create' 或 curl -XPUT 'localhost:9200/bank/account/_create' 返回状态码为201表示创建成功，失败则返回409

2. 批量插入

curl -XPOST 'ad-test.es.byted.org:80/bank/account/_bulk?pretty' --data-binary @accounts.json

  1. _bulk是rest得命令，可以批量执行多个操作

查询

1.基本查询(根据索引，类型，文档的id查找)

内容基本查询

curl -XGET 'ad-test.es.byted.org:80/bank/account/_search?q=Street&pretty'
_search表示搜索，q后面跟着搜索的条件，查询某一字段包含street的文档，q=*表示查询所有的文档。

返回结果中有hits字段，符合查询条件的内容封装在里面，hits.total表明符合查询条件文档的数量，但是默认hits.hits中默认返回10条结果，可以指定size来指定返回结果。

不过建议不在使用上述的请求参数方式，而是采用请求体方式

multi_match 是 match 的作为在多个字段运行相同操作的一个速记法。fields 属性用来指定查询针对的字段, size为返回结果的数量，from表示从第几个结果后开始返回，_source表示返回结果文档的哪些字段， highlight为高亮。

3. Bool 查询

在某些场景下，需要对多个结果进行逻辑操作（与或非等），bool查询接受如下参数：

  • must 等同于 AND
  • must_not 等同于 NOT
  • should 等同于 OR

说明：（地址为北京或上海）&& （年龄为23）&&（名字不为liu）

4. 模糊查询

在进行匹配和多项匹配时，可以启用模糊匹配来捕捉拼写错误，模糊度是基于原始单词的编辑距离来指定的。

5. 通配符查询

常用的通配符有?和*，?匹配任何字符，*匹配零个或多个字符

wildcard表示通配符查询
