spark 源码阅读环境搭建

Posted on 2019-02-27

源码下载

官网https://github.com/apache/spark 可以直接下载（本文最新的是3.0），不过推荐fork到自己的github中，然后再clone到本地，方便以后注释可以push上去。

源码编译

因为源码比较庞大，并且需要先进行编译安装依赖环境和生成一些其他的依赖源码，官网编译文档，http://spark.apache.org/docs/latest/building-spark.html#buildmvn
因为基本都是通过IntelliJ 阅读源码，所以只要运行
./build/mvn -T 4 -DskipTests clean package
其中T是线程数，等待编译完成即刻，大概10多分钟左右。

备注：
1.编译了三次，前2次都是卡在spark-mllib-local_2.12-3.0.0-SNAPSHOT-test-sources.jar就不动了，后面把前面编译-T 多线程参数去掉，重新编译就成功了，具体原因我也不知道。
2.其实不需要提前安装scala，maven，jdk等这些，spark编译包会自动根据pom文件中指定的版本安装。
3.完成后备忘了运行./bin/spark-shell测试一下是否成功了哈。

源码导入intellij

maven工程的打开比较方便
Menu -> File -> Open -> {spark dir}/pom.xml -> Open as Project

运行测试程序

spark中带有各类语言测试程序，在examples文件中，本文以java语言最经典的wordCount程序为例。

命令行运行

官网的说明是use bin/run-example [params] in the top-level Spark directory，即在spark的根目录下运行bin/run-example。然后系统会自动调用spark-submit脚本运行任务。如
./bin/run-example JavaWordCount /Users/liuzebin/open_source_project/spark/examples/src/main/resources/people.txt

idea中直接运行

配置jdk

打开File–>Project Structure–>Platform settings–>SDKS选择Java的安装路径即可Apply即可。
我在mac上安装的jdk11，默认路径为/Library/Java/JavaVirtualMachines/jdk-11.0.1.jdk/Contents/Home

然后这问题解决了，又出来新问题，心累哈

发现最外层的pom中，哎，决定一不做二不休，把pom中的provided全替换为compile

最后终于ok啦，输出了大量的info日志，哈哈，开心。

java.lang.NoClassDefFoundError 基本都是类导入失败，大概是pom文件中对应类的scope为provided,删除或改为compile即可。

单步debug

在程序对应的点打上断点，然后运行debug即刻，然后就可以看到变量的值啦。
